# m1 Tutorial: Interactive Chat + Session Persistence

## Learning Objective

By the end of this milestone, you can:
- run multi-turn chat
- resume prior sessions
- inspect stored transcripts/events

## Starting Point (Rolling)

Use current `main` and run:

```bash
npm run lint
npm test
```

Optional: inspect historical snapshot with `git checkout m1`.

## Why m1 Exists

m0 is one-shot. Real agent workflows need continuity:
- memory across turns
- reproducibility/debugging via logs
- resume after process restart

## What Gets Added

- `nano-agent chat`
- `nano-agent sessions`
- `nano-agent chat --resume <sessionId>`
- local storage under `.nano-agent/sessions/<id>/`

## Important Files

- `src/cli/chat.ts`
- `src/core/session-store.ts`
- `src/cli/list-sessions.ts`

## Storage Layout

For each session:
- `meta.json`: session metadata
- `transcript.jsonl`: user/assistant messages
- `events.jsonl`: runtime event log

## Step-by-Step Walkthrough

1. Start chat
- create a new session ID
- print it to terminal

2. For each user message
- append user message to transcript
- build prompt with recent history
- execute provider run

3. For each runtime event
- render to terminal
- append to `events.jsonl`
- aggregate assistant text and persist

4. Resume
- load session by ID
- continue from prior local transcript

## Run It

```bash
npm run dev -- chat
npm run dev -- sessions
npm run dev -- chat --resume <session-id>
```

## Expected Output Pattern

- session id shown at start
- `sessions` lists IDs + last update times
- resumed chat keeps context continuity

## Troubleshooting

1. `session not found`
- use exact id from `nano-agent sessions`

2. transcript feels stale
- only recent history is injected by design

3. storage confusion
- inspect `.nano-agent/sessions/<id>/` directly

## Exercises

1. Easy: print active session file paths on startup.
2. Medium: add `/history` command to dump last N messages.
3. Hard: implement transcript compaction policy.

# m1 Tutorial - Interactive Chat with Session Persistence

## Goal
Evolve from one-shot tasks to a practical chat workflow with local state.

## What you build
- `nano-agent chat`
- `nano-agent sessions`
- `nano-agent chat --resume <sessionId>`
- Local persistence at `.nano-agent/sessions/<id>/`

## Key files
- `src/cli/chat.ts`
- `src/core/session-store.ts`
- `src/cli/list-sessions.ts`

## Step-by-step
1. Create a session ID and metadata (`meta.json`).
2. Append each user/assistant message to `transcript.jsonl`.
3. Append runtime events to `events.jsonl`.
4. Rebuild prompt context from recent transcript lines.
5. Resume by loading local session metadata/transcript.

## Run it
```bash
npm run dev -- chat
npm run dev -- sessions
npm run dev -- chat --resume <session-id>
```

## Expected behavior
- Session ID shown at chat start.
- Messages continue from prior local history.

## Common issues
- `session not found`: verify ID from `nano-agent sessions`.
- Large prompts: only recent messages are included by design.

## Exercises
- Add transcript compaction for long conversations.
- Store and show token usage per turn.

# m2 Tutorial - Add a Todo Planning Layer

## Goal
Introduce explicit task planning constraints before deeper orchestration.

## What you build
- Session-scoped `TodoManager`
- Chat commands:
  - `/todo add <text>`
  - `/todo start <id>`
  - `/todo done <id>`
  - `/todo list`
- Todo context injected into prompts

## Key files
- `src/core/todo-manager.ts`
- `src/cli/todo.ts`
- `src/cli/chat.ts`

## Constraints implemented
- Maximum 20 todos
- Only one `in_progress` item at a time

## Run it
```bash
npm run dev -- chat
# inside chat
/todo add build rpc client
/todo start 1
/todo list
```

## Expected behavior
- Todo commands update `.nano-agent/sessions/<id>/todo.json`.
- Current todo state is included in each turn prompt.

## Common issues
- `only one in_progress`: mark current active item done before starting another.

## Exercises
- Add dependency fields between todos.
- Add `/todo remove <id>` with safety checks.

# m4 Tutorial: Approvals + In-Flight Turn Control

## Learning Objective

By the end of this milestone, your CLI can actively control running turns in RPC mode:
- approve/deny runtime actions
- steer a turn mid-flight
- interrupt a turn

## Starting Point (Rolling)

Use current `main` and run:

```bash
npm run lint
npm test
```

Optional: inspect historical snapshot with `git checkout m4`.

## Why m4 Exists

Without active controls, the CLI is mostly a viewer.
This milestone turns it into an operator console.

## What Gets Added

- interactive approval prompt in chat (`approve action? [y/N]`)
- `nano-agent steer <sessionId> "<text>"`
- `nano-agent interrupt <sessionId>`
- RPC provider methods:
  - `turn/steer`
  - `turn/interrupt`
  - approval response plumbing

## Important Files

- `src/providers/codex-rpc/provider.ts`
- `src/providers/codex-rpc/jsonrpc-client.ts`
- `src/cli/chat.ts`
- `src/index.ts`

## Step-by-Step Walkthrough

1. Approval handling
- detect approval request notification
- prompt user in terminal
- send response via JSON-RPC

2. Steer command
- send new user input to in-flight turn using `turn/steer`

3. Interrupt command
- request cancellation via `turn/interrupt`

## Run It

```bash
NANO_PROVIDER=rpc npm run dev -- chat
# in another terminal when needed:
NANO_PROVIDER=rpc npm run dev -- steer <sessionId> "focus on writing tests"
NANO_PROVIDER=rpc npm run dev -- interrupt <sessionId>
```

## Expected Output Pattern

- approval prompt appears when RPC requests user approval
- steer command returns `steer sent`
- interrupt command returns `interrupt sent`

## Troubleshooting

1. `steer/interrupt require NANO_PROVIDER=rpc`
- export `NANO_PROVIDER=rpc` before command

2. approval prompt never appears
- current run may not trigger gated actions

3. wrong session id
- use `nano-agent sessions` and match active session

## Exercises

1. Easy: record every approval decision in transcript.
2. Medium: add `--yes` auto-approve flag for trusted environments.
3. Hard: implement approval policy by risk category.

## Recap

At m4, your CLI is a practical Codex controller with:
- stable event model
- persistent sessions
- planning layer
- RPC transport
- operational controls

# m0 Tutorial: First Runnable Codex Control Plane

## Learning Objective

By the end of this milestone, you can run:

```bash
nano-agent run "<task>"
```

and understand how raw Codex JSONL events become app-level runtime events.

## Starting Point

Checkout the checkpoint:

```bash
git checkout m0
npm install
npm run lint
npm test
```

## Why m0 Exists

Before adding chat, planning, or RPC complexity, you need one stable foundation:
- delegate a task to Codex
- stream machine-readable output
- normalize events for your own UI/runtime

## Architecture (m0)

1. CLI parses `run` command.
2. `CodexExecProvider` spawns `codex exec --json`.
3. stdout lines are parsed as JSONL.
4. events are mapped to `RuntimeEvent` union.
5. renderer prints deltas and terminal states.

## Important Files

- `src/index.ts`
- `src/providers/codex-exec/provider.ts`
- `src/providers/codex-exec/jsonl.ts`
- `src/types/events.ts`
- `src/cli/render.ts`

## Step-by-Step Walkthrough

1. `src/index.ts`
- validates CLI command
- forwards task text to provider

2. `provider.ts`
- spawns child process: `codex exec --json <task>`
- streams `stdout` and `stderr`
- yields normalized runtime events

3. `jsonl.ts`
- splits stream into full lines
- parses each line as JSON
- maps transport events to internal types

4. `render.ts`
- prints assistant token deltas
- prints completion/failure markers

## Run It

```bash
npm run dev -- run "summarize this repository"
```

## Expected Output Pattern

You should see:
- incremental assistant text
- final marker: `[turn completed]`

On failure:
- `[error] ...` and/or `[turn failed] ...`

## Troubleshooting

1. `codex: command not found`
- Install Codex CLI and ensure it is in PATH.

2. Empty output for a long time
- wait for Codex startup/network
- inspect stderr lines printed by the provider

3. Immediate non-zero exit
- verify Codex auth and model access outside this project first

## Exercises

1. Easy: count how many events were parsed per run.
2. Medium: write raw JSONL events to `.nano-agent/raw-events.log`.
3. Hard: add a `--json-out <file>` CLI option.

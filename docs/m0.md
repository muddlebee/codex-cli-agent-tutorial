# m0 Tutorial - First Runnable Codex Control Plane

## Goal
Build the smallest useful CLI that can delegate work to Codex using machine-readable events.

## What you build
- `nano-agent run "<task>"`
- A `CodexExecProvider` wrapping `codex exec --json`
- Event normalization (`raw JSONL` -> `RuntimeEvent`)

## Key files
- `src/providers/codex-exec/provider.ts`
- `src/providers/codex-exec/jsonl.ts`
- `src/index.ts`
- `src/types/events.ts`

## Step-by-step
1. Spawn `codex exec --json` using `child_process.spawn`.
2. Read `stdout` as UTF-8 and split by newline.
3. Parse each line as JSON.
4. Map transport events into app-level events (`assistant_delta`, `turn_completed`, `turn_failed`).
5. Stream events to terminal renderer.

## Run it
```bash
npm install
npm run dev -- run "summarize the repository"
```

## Expected behavior
- Assistant text appears incrementally.
- On success, you see `[turn completed]`.
- On execution failure, you see `[turn failed] ...`.

## Common issues
- `codex: command not found`: install Codex CLI first.
- Empty output: command may still be running; check stderr events.

## Exercises
- Add an event counter by event type.
- Log raw events to a file for replay.

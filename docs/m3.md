# m3 Tutorial - Migrate to Codex App-Server RPC

## Goal
Switch from one-shot exec transport to persistent Codex thread/turn APIs.

## What you build
- `JsonRpcClient` over stdio to `codex-app-server`
- `CodexRpcProvider` with:
  - `thread/start`
  - `thread/resume`
  - `turn/start`
- Provider selection via `NANO_PROVIDER=exec|rpc`

## Key files
- `src/providers/codex-rpc/jsonrpc-client.ts`
- `src/providers/codex-rpc/provider.ts`
- `src/providers/factory.ts`

## Step-by-step
1. Spawn `codex-app-server`.
2. Send JSON-RPC `initialize` then `initialized`.
3. Start/resume thread.
4. Start turn and consume notifications.
5. Map notifications into shared `RuntimeEvent` type.

## Run it
```bash
NANO_PROVIDER=rpc npm run dev -- chat
```

## Expected behavior
- Same CLI UX, but backed by persistent RPC thread state.

## Common issues
- `codex-app-server not found`: set `CODEX_APP_SERVER_BIN` if binary path is custom.
- `thread/start did not return a thread id`: verify Codex app-server version compatibility.

## Exercises
- Implement `thread/list` and expose it in CLI.
- Add stronger JSON-RPC schema validation.

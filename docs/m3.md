# m3 - Codex App-Server RPC Provider

## What was added
- JSON-RPC stdio client for `codex-app-server`
- `CodexRpcProvider` with `thread/start`, `thread/resume`, and `turn/start`
- Provider factory using `NANO_PROVIDER=exec|rpc`

## Why this matters
This milestone upgrades the control plane from single-shot exec calls to persistent thread/turn APIs exposed by Codex RPC.
